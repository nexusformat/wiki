Minutes¶

  • FA: NXtextlog for time based text messages. Note that NXnote only has 1 timestamp.

    • MK: NXlog could be generalized to NX_CHAR_OR_NUMBER,

    • FA: yes, it is an option, but NXlog would become no longer plottable

    • PC: NXnote allows textlogs

    • FA: yes, but it would be nice to have timestamps attached to each separate log message

    • SB: NXnote could be extended to host a timestamp array, too.

    • AB: NXtextlog could be a base class of the other logs. I like NXtextlog as proposed

    • BW: +1 (adding something new is more backward compatible and cleaner than modifying an existing and widely used standard)

    • SB: +1

    • AB: Accepted the suggestion

    • SB: PJ suggested to accept it in contributed and when it is used vote for accepting it in the standard)

    • AB: Yes, it should go to contributed without voting, but reviewed, and when it is used in AppDef, we should vote on it.

  • NeXus paper

    • MK: paper is drafted and initial review is done. We are welcoming NIAC contributions

    • AB: I still need to review, NIAC2024 attendees should also be invited, but we should not miss contributions from NIAC, so we should make open for the full NIAC

    • MK: this needs to be cross-checked first

  • NIAC 2026

    • AB: LBNL is volunteering to host it in Q3, alternatively we could be a satellite of NOBUGS in Sept in Hamburg

    • MK: Hamburg would be better suited

    • AB: FdA could be a local host for NIAC2026 attached to NOBUGS

  • NeXus Ontology

    • SB suggestion to move to w3id, but also keep existing purl registry. PR for regenerating the ontology is prepared. Note that PM requested to keep purl registration and update it to point to the new IRI.

    • HG: W3ID is preferred, and we could also gather there all PAN related ontologies.
